# IoT-ThreatLens

Grad project for IoT network intrusion detection. You upload a network capture file (.csv or .parquet), it runs it through a trained XGBoost model, flags malicious traffic, and lets you explore the results on a dashboard. You can also click into any threat and query Gemini to get an explanation of what the traffic actually means.

---

## How it works

The frontend is React, backend is FastAPI. When you upload a file the backend preprocesses it, runs XGBoost, and streams progress back to the browser in real time using SSE. Results stay in memory on the backend and the frontend paginates through them on demand so the browser doesn't choke on large files.

Attack types and severity scores are assigned by a rule-based classifier (not Gemini) based on traffic fingerprints like connection state, protocol, and service. Gemini is only called when you explicitly ask for an explanation on a specific threat.

```
React -> FastAPI -> XGBoost -> Rule-based classifier
                     |
              Gemini (on demand only)
```

## Requirements

- Python 3.10+
- Node.js 18+
- Gemini API key (free tier works)
- HuggingFace account + token (only needed if you want to retrain)

---

## Setup

**1. Environment variables**

Make a `.env` in `Application/`:
```
GEMINI_API_KEY=your_key_here
FRONTEND_URL=http://localhost:3000
HF_TOKEN=your_huggingface_token_here
```
HuggingFace token is only needed if you want to retrain the model. Get one at huggingface.co/settings/tokens.

Make a `.env` in `Application/frontend/`:
```
REACT_APP_API_URL=http://localhost:8000
```

**2. Backend**

```bash
cd Application/backend
pip install -r requirements.txt
uvicorn main:app --reload
```

**3. Frontend**

```bash
cd Application/frontend
npm install
npm start
```

## Model artifacts

The model artifacts (`xgb_model.ubj`, `scaler.pkl`, `encoders.pkl`) are already in `backend/` so it should just work. They were generated by the training notebook. If you want to retrain, run all the cells in `Model - XGBoost.ipynb` — the last cell saves everything back to `backend/`.

## Usage

1. Go to `http://localhost:3000`
2. Upload a `.csv` or `.parquet` file — needs these columns: `proto`, `service`, `conn_state`, `orig_bytes`, `resp_bytes`, `duration`, `orig_pkts`, `resp_pkts`, `missed_bytes`
3. Watch it process in real time
4. Browse threats by severity/attack type on the dashboard
5. Click a threat and hit **Query AI Agent** if you want Gemini to explain it

If you just want to look around without uploading anything, hit **View Demo Dashboard** on the home page.

---

## Dataset

Trained on [IoT-23](https://www.stratosphereips.org/datasets-iot23) from Stratosphere Lab — real malware captures from 23 IoT scenarios plus benign traffic. Got the preprocessed version from HuggingFace (`19kmunz/iot-23-preprocessed`). Balanced to 250k benign / 250k malicious for training.
